# Ares Engine

**Stack**: FastAPI + Neon PostgreSQL + DeepSeek-R1/V3 + Anthropic Claude 3.5 Sonnet + Gemini 2.5 Flash (`gemini-2.5-flash` via `google-genai` SDK) + Exa.ai (Native Tools with domain quality filters) + DataForSEO MCP
**Budget**: $10 max ŌĆö prefer lightweight, serverless

## Key Paths
- `app/main.py` ŌĆö FastAPI app, `/generate` SSE endpoint, `/posts/{id}/approve`, `/clarify`, `/rules`, `/workspaces`, `/campaigns/plan`, `/campaigns`
- `app/services/` ŌĆö All agents: briefing_agent, research_service, source_verification_service, psychology_agent, writer_service, readability_service, feedback_service, research_intel_service, writer_intel_service, cartographer_service
- `app/models.py` ŌĆö ORM models: `Post`, `UserStyleRule`, `ResearchCache`, `Workspace`, `ResearchRun`, `NichePlaybook`, `WriterRun`, `WriterPlaybook`, `ContentCampaign`, `VerifiedSource`, `FactCitation`, `DomainCredibilityCache`
- `app/schemas.py` ŌĆö Pydantic schemas (includes `WorkspaceCreate`, `WorkspaceResponse`, `StyleRuleCreate`, `ResearchRunCapture`, `NichePlaybookResponse`, `CampaignResponse`, `VerifiedSourceResponse`, `FactCitationResponse`)
- `app/services/prompts/` ŌĆö LLM prompt templates (writer.md, persuasion.md)
- `static/` ŌĆö Frontend (ares_console.html, js/console.js, js/api.js)
- `app/database.py` ŌĆö Neon PostgreSQL connection (SQLAlchemy, pool_pre_ping, keepalives)
- `app/settings.py` ŌĆö Environment + API key loading

## Rules
- **Async mandatory** ŌĆö all HTTP clients + generation calls must be async
- **Pydantic-first** ŌĆö validate at every agent boundary via `app/schemas.py`
- **Zero Hallucination** ŌĆö enforce strict tool schemas in ResearchAgent; hallucinated tools trigger an error block to force R1 self-correction
- **Iterative Tooling** ŌĆö ResearchAgent runs an iterative loop (max 5) allowing R1 to mix DataForSEO MCP tools with native Exa tools (`exa_scout_search`, `exa_extract_full_text`). Agentic prompt uses a **3-step sequencing pattern**: Step 1 (mandatory tools with literal JSON example pre-filled with keyword), Step 2 (strategic tools like Exa with quality filters/backlinks/on_page), Step 3 (final output only after Steps 1-2). A `CRITICAL` preamble enforces tool-calling before any final analysis.
- **Exa.ai Domain Quality Filters** ŌĆö Defense-in-depth source credibility via niche-aware domain filtering. `exa_scout_search` accepts `include_domains`, `exclude_domains`, `start_published_date`, `num_results` (default 10, up from 5). `_build_agentic_prompt()` auto-injects authoritative domain lists based on workspace niche: cybersecurity (nist.gov, cisa.gov, owasp.org, schneier.com, etc.), AI/ML (arxiv.org, openai.com, anthropic.com, deepmind.com, etc.), general (.edu, .gov, nature.com, science.org). Auto-excludes low-quality domains (medium.com, blogspot.com, wordpress.com). Enforces 2-year recency filter. R1 receives explicit STEP 2 instructions with domain lists and JSON examples. Reduces Phase 1.5 rejection rate from ~20% to ~5%.
- **Expanded Research Output** ŌĆö R1's final output is an expanded dict with keys: `information_gap`, `unique_angles`, `competitor_weaknesses`, `data_points`, `practitioner_insights`. Legacy string format still supported via fallback. Result dict unpacks these into top-level keys. On-page metrics and backlink authority are extracted from real MCP tool results (no hardcoded placeholders). String responses are auto-parsed via `json.loads` in case R1 returns stringified JSON.
- **Multi-tenant** ŌĆö all DB queries must filter by `profile_name` (workspace scope)
- **Cache Isolation** ŌĆö ResearchCache uses composite unique key `(keyword, profile_name, niche)` to prevent cross-workspace/niche pollution. All cache lookups require exact match on all three fields.
- **Playbook Boundaries** ŌĆö Niche playbooks are injected within `<niche_playbook>` XML tags with explicit instruction to DeepSeek-R1 to use them ONLY for strategic patterns, NOT past topic research.
- **Frontend State Management** ŌĆö Global variables (`lastGeneratedMarkdown`, `currentPostId`, `currentQuestions`) are cleared before each generation to prevent UI artifacts from previous runs.
- **Research Intelligence** ŌĆö ResearchAgent self-improves via a 4-phase loop: Capture ($0) ŌåÆ Recall (~200 tokens) ŌåÆ Reinforce ($0 on /approve) ŌåÆ Distill (~$0.001/10 runs via Gemini Flash). Niche playbooks are scoped by `(profile_name, niche)`.
- **Writer Intelligence** ŌĆö WriterService self-improves via parallel 4-phase loop: Capture ($0) ŌåÆ Recall (~100 tokens) ŌåÆ Reinforce ($0 on /approve) ŌåÆ Distill (~$0.001/10 runs, heuristic). WriterRun captures readability telemetry (ARI, FK, CLI, sentence length) for each article. WriterPlaybook distills niche-specific patterns (avg ARI baseline, target sentence length, structure templates) when Ōēź10 approved runs + Ōēź5 quality scores accumulate. Readability efficiency formula: `(10.0 - ARI) / 10.0`. Only runs with efficiency Ōēź0.20 (ARI Ōēż8.0) included in distillation. Playbooks scoped by `(profile_name, niche)`. Complements UserStyleRule (human preferences) by learning what achieves readability targets.
- **Source Verification (Phase 1.5)** ŌĆö SourceVerificationService validates credibility of Exa.ai sources before article generation using `gemini-2.5-flash` via async `google-genai` SDK. Uses `client.aio.models.generate_content()` with `response_mime_type="application/json"` for reliable JSON responses and robust error handling with fallback scores. **URL deduplication** at verification start prevents database constraint violations when research returns duplicate URLs. DataForSEO integration uses defensive MCP response parsing (supports multiple formats: `.content[0].text`, `.text`, fallback), field validation before int casting, and detailed debug logging. **Rebalanced multi-factor scoring (0-100)** (March 2026): Content Quality via Gemini Flash (40pts), Domain Tier via curated lists (30pts), Content Freshness (20pts), Internal Citations (10pts). Minimum threshold: 60/100 to pass. Generation **fails** if <3 credible sources found. **Enhanced date extraction** scans 5000 chars (was 2000) with 10 regex patterns (was 4) including ISO 8601, "Updated:", "Last modified:", URL path fallback. Extracts factual claims (stats, benchmarks, case studies, expert quotes) via **improved Gemini Flash prompt** with explicit examples and confidence scoring (Ōēź0.6 required, lowered from 0.7). **Expanded quantitative claim detection** uses 8 regex patterns (was 4): percentages, dollars, statistics, benchmarks, spelled-out fractions, narrative claims, comparatives, year-based stats. **Relaxed citation validation** accepts 3 formats: markdown links [text](url), parenthetical (Source 2024), footnotes [1]. Injects citation map into WriterService prompt with **critical emphasis** (visual borders, explicit examples, strict prohibitions). Triple-gate validation: SEO ŌåÆ Citations (intelligent, based on quantitative claims) ŌåÆ Readability. Citation requirements: articles with quantitative claims (stats, %, $, benchmarks) require min 3 citations; articles with 3+ claims require 1 citation per claim; general advice articles with no quantitative claims require 0 citations (validation skipped). Cost: ~$0.011/article first run, ~$0.001/article after caching. All tables scoped by `(profile_name, niche)`.
- **Readability Enforcement** ŌĆö WriterService enforces 7th-10th grade readability (target Ōēż10.0, accepts 7.5-10 range) via triple-gate validation: SEO structure first, citations second (intelligent requirements based on quantitative claims), then composite readability scoring (ARI primary, Flesch-Kincaid cross-check with +1.5 buffer, Coleman-Liau advisory only). Max 5 iterative rewrites. Zero API cost for readability (Gemini Flash only for citation extraction in Phase 1.5). READABILITY_DIRECTIVE injected dynamically (never modifies writer.md). Includes **pre-flight simplicity primer** before main directive and **7th-grade template sentences** for pattern-matching. Directive uses **layer-cake scanning format** optimized for how busy readers scan (headings ŌåÆ first sentences ŌåÆ bold text). Requires benefit-driven H2s every 150-200 words, key takeaway as first sentence of each section, and bold anchor phrase per section. Target word count: 1,500-1,800 words. Bans AI-slop words directly in the directive. Includes a concrete **word-swap reference table** (implementŌåÆset up, utilizeŌåÆuse, demonstrateŌåÆshow, streamlineŌåÆsimplify, leverageŌåÆusing, etc.) repeated in feedback on every iteration so Claude has explicit short-word alternatives. Enforces 8-12 words/sentence (MANDATORY for 80% of sentences, never exceed 15 words). **80% distribution gate**: requires Ōēź80% of sentences to fall in 8-12 word range (not just average). **Complex sentence gate**: Ōēż15% of sentences can exceed 15 words (reduced from 20%). **Distribution feedback**: shows exact breakdown ("Only 62% in 8-12 range, need 80%") with actionable rebalancing steps. **Broad keyword masking**: scoring masks semantic keywords + blueprint entities + 58+ niche terms including business jargon (security, business, software, streamline, leverage, optimize, enhance, framework, methodology, ecosystem, paradigm, etc.) that inflate ARI but have no shorter synonym. **Citation masking**: inline citations treated as single words during scoring (URLs stripped). **Readability tracking**: scores (ARI, FK, CLI, avg sentence length) persisted to Post.readability_score JSON column for analytics. SEO failure feedback reports all 6 validation conditions with specific counts (word count, H1, H2, list/table blocks, info gain density, banned words).
- **No Fake Assets & No Fabricated Data** ŌĆö WriterService prompt includes hard constraints (constraints 5 and 6) banning references to non-existent templates, tools, downloads, checklists, or frameworks, and strictly forbidding the fabrication of statistics, percentages, or dollar amounts. The model must give actionable steps and use ONLY verified facts from the citation map provided (extracted from credible sources in Phase 1.5).
- **SSL Retry on Commit** ŌĆö Post-generation `db.commit()` in `event_generator()` is wrapped in `OperationalError` retry that gets a fresh `SessionLocal()` if Neon drops the SSL connection during long writer loops. Uses `nonlocal db` to reassign the outer scope's session. Only applied to post-generation commits, not every commit in the file.
- **API keys** ŌĆö ANTHROPIC_API_KEY, GEMINI_API_KEY, DEEPSEEK_API_KEY, EXA_API_KEY, DATAFORSEO_LOGIN, DATAFORSEO_PASSWORD (in .env only)
